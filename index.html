HTML

  • HTML is used to create the structure of a web page and is made up of elements which work together to create the basic layout and framework of the page.
  • The head element contains information about the webpage.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
  • Most HTML elements have an opening and a closing tag.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are 3 ways to style a website using CSS: Inline CSS, Internal Style Sheets and External Style Sheets.
  • * is used to apply a rule to all elements visible on the page.
  • You can select individual elements to style using their name e.g. h1, h2 etc.
  • Two or more elements that will share the same styling can be grouped together to help write DRY (Do Not Repeat Yourself) code.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds all changes to the next commit.
  • git commit -m "insert message here": commits the changes with the commit message.
  • git pull origin main: checks our local branch is in sync with the base branch in GitHub.
  • git push origin branch-name: pushes our changes to the remote branch
